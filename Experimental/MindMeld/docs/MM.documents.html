Namespace: documents

MM. documents

MM.documents represents the whole Documents collection that are part of a particular application. These Documents are not related to a particular Session.

Methods

delete(documentid, onSuccess, onFail)

Delete a document from the application. This requires an admin token

Name Type Description
documentid string

id of the document to delete
onSuccess APISuccessCallback optional

callback for when deleting object was successful
onFail APIErrorCallback optional

callback for when deleting object failed
Example
         function deleteDocument () {
            MM.documents.delete('381c21d853faf6db58a0ab7d7d12e604', onDocumentDeleted);
         }
         function onDocumentDeleted (response) {
            // document with documentid response.data.documentid deleted
         }

get(params, onSuccess, onFail)

Get and search across all documents indexed for your application. This endpoint will let you access all documents that have been crawled from your website as well as all documents that you have posted to the documents collection for this application. User privileges do not permit access to this object; admin privileges are required

Name Type Description
params QueryParameters optional

A QueryParameters object allowing you to filter the documents returned. See documentation here for more details. For this function, the following additional parameters are also available:

Name Type Description
document-ranking-factors string optional

A JSON string containing custom factors that will be used to rank the documents returned by this request. Read the section on custom ranking factors to learn more about how you can adjust the search ranking factors to customize the document results for your application
onSuccess APISuccessCallback optional

callback for when getting the application document list was successful
onFail APIErrorCallback optional

callback for when getting the application document list failed
Example
         function getDocuments () {
            var queryParams = {
                query: 'san francisco' // get documents matching the string 'san francisco'
            };
            // add custom ranking factors to this query
            queryParams['document-ranking-factors'] = {
                recency: 0.5,
                popularity: 0.7,
                relevance: 0,
                proximity: 0.2,
                customrank1: 0.3
            };
            MM.documents.get(queryParams, onGetDocuments);
         }
         function onGetDocuments (response) {
            var documents = response.data;
         }

json(){Array.<Object>}

Helper function returns the JSON data from the application's document collection

Example
         function getDocuments () {
            MM.documents.onUpdate(onGetDocuments);
            MM.documents.get();
         }
         function onGetDocuments () {
            var documents = MM.documents.json();
         }

onUpdate(updateHandler)

Sets the MM documents' onUpdate handler. Pass null as the updateHandler parameter to deregister a previously set updateHandler. Note that there are no push events for the documents collection so it must be polled instead. The update handler will be called automatically when calling MM.documents#get

Name Type Description
updateHandler APISuccessCallback optional

callback for when the app's document list updates.
Example
         function getDocuments () {
            MM.documents.onUpdate(onGetDocuments);
            MM.documents.get();
         }
         function onGetDocuments () {
            var documents = MM.documents.json();
            console.log(documents);
         }

post(document, onSuccess, onFail)

Upload a document to the application. This requires an admin token

Name Type Description
document Object

object containing document data. The only required parameters are 'title' and 'originurl'. Please see Document documentation here for more info

Name Type Description
title string

The title of the document
originurl string

The fully qualified link to the webpage containing the original document. Note that this url will be stored, but not returned in subsequent GET requests to this document object. Instead, the 'originurl' field value will contain a wrapper url which, when loaded in a browser, will record a page view for this document and then redirect to the originurl value provided here. This mechanism enables the view count to be tracked and used to influence the document ranking calculation
description string optional

A short text description of the contents of the document
text string optional

The full text contents of the document
sections string optional

The text from the header sections of the document. This includes any text contained in the h1, h2, h3, h4 and h5 tags, if your document is a webpage
pubdate number optional

The Unix timestamp reflecting the date when this document was originally published
language string optional

The 3-letter ISO-639-2 language code indicating the language of this document (e.g. 'eng', 'spa', 'ger', etc.)
image Object optional

An object specifying information about an image related to the document

Name Type Description
url string optional

The URL for the image associated with this document
thumburl string optional

The URL for a small-format image, if available. This is typically a thumbnail of a larger image and it should have a maximum dimension of around 500 pixels or less
width number optional

The width of the full image in pixels
height number optional

The height of the full image in pixels
source Object optional

An object specifying information about the source of this document

Name Type Description
name string optional

A text string suitable for display containing the name of the source of the document (e.g. 'The New York Times')
url string optional

The website for the document source homepage (e.g. 'www.nyt.com')
icon string optional

The url for an icon, typically a favicon, representing the source
location Location optional

The location associated with this document
customrank1 number optional

A custom numerical rank value that can be used in the document ranking calculation. See the documentation on custom ranking factors for more information
customrank2 number optional

A custom numerical rank value that can be used in the document ranking calculation. See the documentation on custom ranking factors for more information
customrank3 number optional

A custom numerical rank value that can be used in the document ranking calculation. See the documentation on custom ranking factors for more information
onSuccess APISuccessCallback optional

callback for when posting data to collection was successful
onFail APIErrorCallback optional

callback for when posting data to collection failed
Example
         function addDocument () {
            var newDocumentData = {
                title: 'new document title',
                originurl: 'www.expectlabs.com'
            };
            MM.documents.post(newDocumentData, onDocumentAdded);
         }
         function onDocumentAdded () {
            // new document added
         }
